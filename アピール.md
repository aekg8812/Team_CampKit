# サボり癖クリア ── アピール用文書

## ひとことコンセプト

> 自分が選んだサボり癖と、何日向き合えているか。それが可視化される習慣化アプリ。

毎日ひとつ、自分の弱点に向き合って課題をこなす。続けるほど記録が育ち、レベルが上がり、成功率が見える。「今日のサボりを裁く」一発ネタではなく、明日も開きたくなる仕組みを作りました。

## 何ができるアプリか

ユーザーはまず、治したいサボり癖をチェックリストから選びます。スマホ依存、部屋の汚さ、運動不足、先延ばし、連絡の後回し、早起き、食生活。選んだものだけが毎日の出題対象になります。

毎日アプリを開くと、選んだ癖の中からランダムで1つが選ばれ、それに関する質問に答えます。回答が終わるとガチャ演出で今日の課題が決まり、カウントダウンタイマーの中でその課題に挑戦します。達成したら写真かコメントで証拠を出して完了。記録が1つ積み上がります。

達成するたびにその分野のレベルが上がり、課題は少しずつ手強くなります。失敗すれば連続記録は途切れ、レベルは据え置き。マイページには成功率の円グラフ、分野ごとの連続記録、これまでの履歴が並びます。

## 技術的なアピールポイント

### 1. 「絶対に動く」ための二段構え設計

ハッカソンでありがちな「本番でAPIが落ちて動かない」を構造で防ぎました。データ保存も画像判定も、外部サービスを使うかどうかを設定スイッチ1つで切り替えられます。

普段の開発とデモはローカル保存・モック判定で完結するため、APIキーが無くても全機能が動きます。発表のときだけスイッチをオンにして、Firebaseのクラウド保存とClaude APIの画像判定を見せる。開発がAPIトラブルで止まらず、コストもかからない構成です。

### 2. データ層の抽象化

画面のコードは「統一ストア」だけを呼びます。その裏でローカル保存かFirebaseかを振り分けるため、保存先を切り替えても画面側のコードは一切変わりません。同じデータ構造をlocalStorageとFirestoreの両方で扱える設計にしました。

### 3. Claude APIによる画像判定

課題の証拠写真を、厳密採点ではなく「明らかにサボっていないか」のゆるい判定にしました。判定がグレーなら合格にし、判定に失敗しても救済で受理します。続けることを邪魔しない、現実的な判定方針です。

### 4. 続けたくなる心理設計

分野ごとの連続記録とレベル制で、「途切れさせたくない」「次はもっと難しいことに挑みたい」という気持ちを引き出します。成功率の可視化が、自分の成長を実感させます。

## ユーザーへの価値

自分で選んだ弱点に毎日向き合い、その軌跡が数字と記録で残ります。何日続いたか、成功率はどれくらいか、どこまでレベルが上がったか。これは自己管理の証跡そのものであり、就活のエピソードや自己PRの素材にもそのまま使えます。

## 技術スタック

React（Vite）、Tailwind CSS、Framer Motion、Howler.js でフロントを構築。クラウド保存に Firebase（Authentication・Firestore）、画像判定に Claude API を使用。成功率の円グラフはライブラリを使わずSVGで自作しました。デプロイは Vercel。

## このあと伸ばせる余地

今回は機能ロジックを固めることを優先し、実績バッジ、スクショ映えする成績表、無料枠の回数制限、有料会員、サボりタイプ診断、そして裁判所をモチーフにした世界観の付与を、追加できる形で設計に織り込んでいます。土台が「絶対に動く」状態でできているので、ここから演出や世界観を載せて、驚きの要素を足していけます。
